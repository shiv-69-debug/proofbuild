Why it matters

Git history is not a build time capsule.

01 / PROVENANCE

Capture the evidence

Source hashes, Git state, test output, manifests, and optional AI build logs travel together.

02 / VERIFICATION

Detect every change

Compare the current project, local archive, or remotely retrieved capsule against its receipt.

03 / RECOVERY

Restore from Filecoin

Delete the local capsule and ProofBuild can retrieve, verify, and restore the Filecoin copy.

Product flow

Filecoin is visible at every critical step.

1 · SNAPSHOT

Hash project files and record build provenance.

2 · PUBLISH

Upload the capsule and retain provider transaction evidence.

3 · VERIFY

Compare remote bytes and query live PDP proof status.

4 · RESTORE

Recover the exact verified project from Filecoin.
